# Example: Frontend Goal

## Objective

Redesign the pricing page for clearer plan comparison and better mobile behavior.

## Acceptance criteria

1. Plans are easy to compare.
2. Primary CTA is visually clear.
3. Page works at mobile, tablet, and desktop widths.
4. Color contrast is accessible.
5. No checkout or billing logic changes.

## Suggested route

`frontend_only` route:

1. clarify;
2. design exploration if direction is unclear;
3. implementation;
4. responsive probes;
5. accessibility checks;
6. design review;
7. ship.

## Hard fails

- Billing logic changes.
- Mobile layout is unusable.
- Contrast fails accessibility threshold.
